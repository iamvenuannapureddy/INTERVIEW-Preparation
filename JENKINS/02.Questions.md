### **1. What is Jenkins, and Why is it Used?**  
âœ… **Jenkins** is an open-source **automation tool** used for **Continuous Integration (CI) and Continuous Deployment (CD)**.  

âœ… **Why Use Jenkins?**

### **2. What Are the Main Components of Jenkins?**  
âœ… **Key Components:**  
- **Jenkins Master** â†’ Manages jobs, schedules builds, and distributes tasks.  
- **Jenkins Agent** â†’ Executes jobs assigned by the master.  
- **Executor** â†’ Runs individual build steps on an agent.

- **Pipeline** â†’ Scripted automation workflow for CI/CD.  
- **Plugins** â†’ Extend Jenkins functionality (e.g., Git, Docker, Kubernetes).

### **3. How Does Jenkins Differ from GitLab CI, Bamboo, and CircleCI?**  

| Feature        | **Jenkins** | **GitLab CI** | **CircleCI** | **Bamboo** |
|---------------|------------|--------------|-------------|------------|
| **License** | Open-source | Built into GitLab | Cloud & Self-Hosted | Paid |

| **Best For** | Custom workflows, large teams | Git-based workflows | Quick cloud CI/CD | Enterprise-level CI/CD |

### **4. Explain Jenkins Architecture (Master, Agent, Executor).**  
âœ… **Jenkins Master-Agent Architecture:**  
- **Master Node** â†’ Manages jobs, schedules builds, and distributes tasks.  
- **Agent Node** â†’ Executes assigned tasks (can be on a separate server).  
- **Executor** â†’ A thread running on an agent that executes build steps.  

âœ… **Example Workflow:**  
1. The **Master** schedules a build job.  
2. The **Agent** executes the job using available **Executors**.  
3. The **Build Results** are sent back to the **Master** for reporting.  

âœ… **Why Use Master-Agent?**  
- **Improves performance by distributing workloads**.  
- **Allows running builds on different OS/platforms**.

### **8. What is a Jenkins Pipeline? How Does It Work?**

âœ… **Example Declarative Pipeline:**  
```groovy

    agent any
    stages {

            steps {

            }

    }

âœ… **How It Works:**

2. It **executes build, test, and deploy stages**.

âœ… **Why Use Pipelines?**  
- **Better version control (stored in Git)**.

    agent any

        stage('Build') {

                sh 'mvn clean package'
            }

}
```

âœ… **Scripted Pipeline Example:**  
```groovy

    stage('Build') {
        sh 'mvn clean package'
    }
}
```

âœ… **Why Use Declarative?**  
- **Easier to read and maintain**.  
- **Built-in error handling**.  

âœ… **Why Use Scripted?**  
- **More flexible for complex workflows**.

âœ… **Why Use Them?**  
- **`input` is for approvals, `parameters` is for customization**.

### **3. How Do You Configure Jenkins for Blue-Green and Canary Deployments?**

      http:
        paths:
          - path: /

              service:
                name: my-app-blue

```

âœ… **Canary Deployment:**

âœ… **Example Canary Deployment with Istio:**

        - destination:
            host: my-app-v1

        - destination:
            host: my-app-v2
          weight: 20
```

âœ… **Why Use It?**  
- **Reduces downtime and risk in deployments**.
